# خوارزمية كشف الملفات المكررة في dupeGuru

## نظرة عامة

**dupeGuru** هو أداة عبر الأنصات (Linux, macOS, Windows) تحدد الملفات المكررة باستخدام استراتيجيات مسح ذكية متعددة. تستخدم الخوارزمية نهجاً متعدد المراحل يجمع بين مقارنة البيانات الوصفية (Metadata)، تجزئة المحتوى (Content Hashing)، والمطابقة الضبابية (Fuzzy Matching) للكشف الدقيق عن المكررات مع الحفاظ على أداء عالي.

## معمارية النظام

يتم تنظيم نظام كشف المكررات إلى ثلاثة مكونات رئيسية:

### 1. **فئة الماسح** (Scanner Class) - `core/scanner.py`

```
الوظائف الرئيسية:
├─ نقطة البداية لعملية المسح
├─ تكوين معاملات المسح والخيارات
├─ تنسيق سير العمل الكامل
└─ معالجة تشكيل المجموعات والتحسين
```

**المسؤول عن:**
- استدعاء محرك المطابقة
- تطبيق المرشحات
- تجميع النتائج
- اختيار الملفات المرجعية

### 2. **وحدة المحرك** (Engine Module) - `core/engine.py`

```
الوظائف الأساسية:
├─ خوارزميات المطابقة الأساسية
├─ استخراج الكلمات ومقارنتها
├─ بناء قواميس الكلمات
├─ المطابقة الضبابية
└─ مقارنة المحتوى القائمة على التجزئة
```

**المسؤول عن:**
- استخراج الكلمات من الأسماء والبيانات
- مقارنة الملفات بناءً على التشابه
- حساب النسب المئوية للمطابقة
- مقارنة التجزئة للملفات المتطابقة تماماً

### 3. **وحدة نظام الملفات** (File System Module) - `core/fs.py`

```
المسؤول عن:
├─ إدارة بيانات الملف (الحجم، التجزئة، المسارات)
├─ تخزين مؤقت لقيم التجزئة
└─ التحقق من وجود الملفات
```

## أنواع المسح

يدعم dupeGuru عدة طرق مسح، كل منها مُحسّنة لحالات استخدام مختلفة:

| نوع المسح | الغرض | طريقة المقارنة | مطابقة دقيقة |
|-----------|-------|---------------|-----------|
| `FILENAME` | كشف أسماء متشابهة | مطابقة كلمات ضبابية | لا |
| `FIELDS` | مطابقة البيانات المنظمة (فنان-عنوان-ألبوم) | مطابقة قائمة على الحقول | لا |
| `FIELDSNOORDER` | مطابقة الحقول بدون اعتبار الترتيب | مطابقة مستقلة عن الترتيب | لا |
| `TAG` | مقارنة علامات الموسيقى/الوسائط | استخراج العلامات + مطابقة ضبابية | ��ا |
| `FOLDERS` | تحديد هياكل المجلدات المكررة | مطابقة المسار الهرمي | لا |
| `CONTENTS` | البحث عن ملفات متطابقة محتوى (تماماً) | مقارنة التجزئة | نعم |
| `FUZZYBLOCK` | البحث عن صور متشابهة (نسخة الصور) | تحليل الصور | لا |
| `EXIFTIMESTAMP` | مطابقة الصور حسب تاريخ EXIF | مقارنة الطابع الزمني | لا |

## استراتيجيات المطابقة الأساسية

### أ. المطابقة بناءً على المحتوى (المكررات الدقيقة)

**الدالة:** `engine.getmatches_by_contents(files, bigsize=0, j=job.nulljob)`

#### مراحل العملية:

```
المرحلة 1: تجميع حسب حجم الملف
├─ فقط الملفات بحجم متطابق يمكن أن تكون مكررات
└─ تحجيم الملفات إلى مجموعات حسب الحجم

المرحلة 2: للمجموعات بـ 2+ ملفات
├─ المرور الأول: مقارنة التجزئة الجزئية
│  ├─ تجزئة أول 4 كيلوبايت (قابل للتكوين)
│  ├─ تجزئة آخر 4 كيلوبايت (قابل للتكوين)
│  └─ إذا اختلفت → الملفات مختلفة ✗
│
├─ المرور الثاني (إذا كان bigsize > 0 و الملف > bigsize):
│  ├─ حساب تجزئة العينة (كتل من المنتصف)
│  └─ إذا اختلفت → الملفات مختلفة ✗
│
└─ المرور الثالث: مقارنة تجزئة المحتوى الكامل
   ├─ حساب SHA-1 الكامل
   └─ إذا تطابقت → 100% مطابقة ✓

النتيجة: Match(file1, file2, 100%)
```

#### استراتيجية التحسين:

```
المستوى 1 (سريع جداً):    O(1)  - مقارنة الحجم فقط
المستوى 2 (وسيط):        O(log n) - تجزئة جزئية
المستوى 3 (بطيء):        O(n)  - تجزئة كاملة
```

**مثال على العتبات:**
```python
حجم الكتلة = 1024 * 1024  # 1 ميجا بايت
الحد الأدنى للملف = 3 * 1024 * 1024  # 3 ميجا (قبل استخدام التجزئة الجزئية)
إزاحة التجزئة الجزئية = (0x4000, 0x4000)  # 16 كيلوبايت من البداية والنهاية
```

### ب. المطابقة بناءً على الاسم/البيانات الوصفية (المكررات الضبابية)

**الدالة:** `engine.getmatches(objects, min_match_percentage=80, ...)`

#### الخطوة 1: استخراج الكلمات

**الدوال:** `getwords(s)` و `getfields(s)`

```python
def getwords(s):
    """
    خطوات المعالجة:
    
    1. تطبيع Unicode (NFD decomposition)
       مثال: "Café" → "Cafe`" (مفكك)
    
    2. استبدال الأحرف الخاصة بمسافات
       مثال: "اسم_الملف-v2.txt" → "اسم الملف v2 txt"
    
    3. التصفية إلى ASCII/أرقام/مسافات
       (مع دعم Unicode > 894)
    
    4. التقسيم على المسافات وإزالة العناصر الفارغة
       مثال: "اسم  الملف" → ["اسم", "الملف"]
    
    النتيجة: قائمة الكلمات
    """
```

#### الخطوة 2: بناء قاموس الكلمات

**الدالة:** `build_word_dict(objects, j=job.nulljob)`

```
الإدخال:
[
  الملف1(كلمات=["صورة", "ألبوم", "2024"]),
  الملف2(كلمات=["صورة", "ألبوم", "2023"]),
  الملف3(كلمات=["صورة", "مجموعة", "2024"])
]

المعالجة:
لكل ملف وكل كلمة:
  └─ أضف الملف إلى مجموعة الكلمة

الإخراج:
{
  "صورة": {الملف1, الملف2, الملف3},
  "ألبوم": {الملف1, الملف2},
  "2024": {الملف1, الملف3},
  "2023": {الملف2},
  "مجموعة": {الملف3}
}
```

#### الخطوة 3: تقليل الكلمات الشائعة

**الدالة:** `reduce_common_words(word_dict, threshold=50)`

**الهدف:** إزالة الكلمات الشائعة جداً التي تظهر في 50+ ملف (نتائج إيجابية كاذبة)

```
الاستراتيجية:
لكل كلمة شائعة:
  ├─ تحقق إذا كان أي ملف لديه فقط كلمات غير شائعة
  ├─ إذا نعم → احتفظ بالكلمة (وإلا لن تتطابق الملفات)
  └─ إذا لا → احذف الكلمة من القاموس

مثال:
كلمة "و" تظهر في 500 ملف → عادة يتم حذفها
لكن إذا كان الملف A يحتوي فقط على كلمة "و" → احتفظ بها
(وإلا لن يكون لدى هذا الملف مطابقات)
```

#### الخطوة 4: دمج الكلمات المتشابهة (اختياري)

**الدالة:** `merge_similar_words(word_dict)`

**الهدف:** تجميع الأخطاء الإملائية والاختلافات الإملائية

```
الخوارزمية:
لكل كلمة في القاموس المرتب (الأقصر أولاً):
  ├─ ابحث عن كلمات متشابهة (تشابه 0.8 كحد أدنى)
  ├─ ادمج جميع الملفات من الكلمات المتشابهة
  └─ احذف إدخالات الكلمات المكررة

مثال:
"صورة" + "صوره" + "صره" → دمج تحت "صره" (الأقصر)
(جميع الملفات مدرجة تحت الكلمة الأقصر)
```

#### الخطوة 5: مقارنة الملفات باستخدام مطابقة الكلمات

**الدالة:** `compare(first, second, flags=())`

**الخوارزمية:**
```
الإدخال:
الملف1: ["فيلم", "حركة", "2024", "HD"]
الملف2: ["فيلم", "حركة", "2023"]

المعالجة:
1. أنشئ نسخة من كلمات الملف الثاني (قابلة للتعديل)
   نسخة_الثاني = ["فيلم", "حركة", "2023"]

2. لكل كلمة في الملف الأول:
   ├─ "فيلم": موجودة في نسخة_الثاني → مطابقة! احذفها ✓
   ├─ "حركة": موجودة في نسخة_الثاني → مطابقة! احذفها ✓
   ├─ "2024": غير موجودة → بدون مطابقة ✗
   └─ "HD": غير موجودة → بدون مطابقة ✗

3. حساب النسبة المئوية:
   عدد المتطابقات = 2
   إجمالي الكلمات = (4 + 3) = 7
   النسبة = (2 × 2) / 7 × 100 = 57.1% ≈ 57%

الإخراج: 57% مطابقة
```

**أعلام المقارنة:**

| العلم | التأثير |
|------|--------|
| `WEIGHT_WORDS` | الكلمات الأطول تستحق أكثر (كلمة 3 أحرف = 3x وزن) |
| `MATCH_SIMILAR_WORDS` | تطبيق المطابقة الضبابية داخل مقارنة الكلمات (الأخطاء تتطابق) |
| `NO_FIELD_ORDER` | مطابقة الحقول بأي ترتيب |

#### الخطوة 6: إجراء المقارنة الشاملة

**الخوارزمية:**
```
الحلقة الرئيسية - معالجة قاموس الكلمات:

بينما يوجد كلمات في القاموس:
  │
  ├─ أخرج كلمة من القاموس
  ├─ احصل على جميع الملفات التي تحتوي على هذه الكلمة
  │
  ├─ لكل ملف في المجموعة:
  │  ├─ قارنه مع جميع الملفات الأخرى في المجموعة
  │  ├─ تخطّ الأزواج المقارنة مسبقاً (تخزين مؤقت)
  │  ├─ حساب نسبة المطابقة
  │  └─ إذا ≥ أدنى نسبة مطابقة → أضفها للنتائج
  │
  └─ انتقل إلى الكلمة التالية

تحسين الذاكرة:
  ├─ أخرج العناصر المعالجة من القاموس (تحرير الذاكرة)
  ├─ تتبع الأزواج المقارنة لتجنب المقارنات المكررة
  ├─ حد أقصى للنتائج: 5,000,000 مطابقة
  └─ معالجة MemoryError بشكل حسن (إرجاع نتائج جزئية)
```

**تحليل التعقيد:**
- **الوقت:** O(W × F²) حيث W = كلمات فريدة، F = ملفات لكل كلمة
- **الفضاء:** O(W × F) لقاموس الكلمات
- **عملياً:** معظم الملفات تحتوي على < 50 كلمة، مما يبقي البحث سريعاً

## تفاصيل التنفيذ

### فئة Match

```python
class Match(namedtuple("Match", "first second percentage")):
    """
    تمثل زوج ملفات متطابق.
    
    الخصائص:
        first: كائن الملف (الملف الأول في الزوج)
        second: كائن الملف (الملف الثاني في الزوج)
        percentage: int 0-100 يشير إلى مستوى المطابقة
                    (100 = مطابقة محتوى دقيقة، <100 = مطابقة ضبابية)
    """
```

### فئة Group

**الدالة:** `get_groups(matches)`

```python
class Group:
    """تدير ملفات متعددة متطابقة معاً."""
    
    # الخصائص الأساسية:
    ref: File           # الملف المرجعي (لن يتم حذفه)
    ordered: List[File] # جميع الملفات مرتبة حسب الأولوية
    unordered: Set[File] # نفس الملفات كمجموعة غير مرتبة
    dupes: List[File]  # ordered[1:] (ملفات قد تُحذف)
    matches: Set[Match] # جميع أزواج المطابقة في المجموعة
    percentage: int    # متوسط نسبة المطابقة مع ref
    
    # الدوال الرئيسية:
    add_match(match)                    # إضافة زوج مطابقة، التحقق من التوافق
    switch_ref(new_ref)                 # تغيير ملف المرجع
    prioritize(key_func, tie_breaker)  # إعادة الترتيب حسب المعايير
    discard_matches()                   # تحرير الذاكرة بعد التجميع
```

#### خوارزمية تشكيل المجموعات

```python
def get_groups(matches):
    """تحويل أزواج المطابقة إلى مجموعات منسجمة."""
    
    matches.sort(key=lambda m: -m.percentage)  # الأعلى أولاً
    
    dupe2group = {}  # تتبع أي مجموعة ينتمي إليها كل ملف
    groups = []
    
    for match in matches:
        first, second, _ = match
        
        # التحقق إذا كانت الملفات موجودة بالفعل في مجموعات
        first_group = dupe2group.get(first)
        second_group = dupe2group.get(second)
        
        if first_group and second_group:
            # كلاهما في مجموعات
            if first_group is second_group:
                # نفس المجموعة → أضف المطابقة
                target_group = first_group
            else:
                # مجموعات مختلفة → تخطّ (ينتهك الانتقالية)
                continue
        elif first_group:
            # الأول في مجموعة، الثاني لا → أضف الثاني
            target_group = first_group
            dupe2group[second] = target_group
        elif second_group:
            # الثاني في مجموعة، الأول لا → أضف الأول
            target_group = second_group
            dupe2group[first] = target_group
        else:
            # لا أحد في مجموعة → أنشئ مجموعة جديدة
            target_group = Group()
            groups.append(target_group)
            dupe2group[first] = target_group
            dupe2group[second] = target_group
        
        # أضف المطابقة إلى المجموعة المستهدفة
        target_group.add_match(match)
    
    # معالجة المطابقات اليتيمة (الموجودة لاحقاً) بشكل متكرر
    return groups
```

## تحسين الأداء

### 1. **مقارنة محتوى متعددة المستويات**

```
حجم الملف < 3 ميجابايت:
  └─ مقارنة التجزئة الكاملة (سريعة بما يكفي)

حجم الملف ≥ 3 ميجابايت:
  ├─ التجزئة الجزئية (16 كيلوبايت البداية + 16 كيلوبايت النهاية)
  ├─ إذا كانت مختلفة → الملفات مختلفة (تخطّ التجزئة الكاملة)
  └─ إذا كانت متطابقة → تجزئة العينة → التجزئة الكاملة (إذا لزم)
```

### 2. **تحسين قاموس الكلمات**

```
إدارة الذاكرة:
  ├─ بناء القاموس بشكل تدريجي
  ├─ أخرج العناصر المعالجة (تحرير الذاكرة)
  ├─ حد أقصى للنتائج: 5,000,000 مطابقة
  └─ معالجة MemoryError بشكل حسن
```

### 3. **آليات إزالة التكرار**

```
تتبع الملفات المقارنة:
  compared = {
    file1: {file2, file3, file5},  # قارن بالفعل مع هؤلاء
    file2: {file1, file4},
    ...
  }
  
يمنع المقارنات الزائدة O(n²)
```

### 4. **التخزين المؤقت**

## اختيار الملف المرجعي

### قواعس الربط

عندما يكون لعدة ملفات نفس الحجم، يطبق dupeGuru هذه القواعد بالترتيب:

```python
def _tie_breaker(ref, dupe):
    """
    إرجاع True إذا كان يجب أن يصبح dupe مرجعاً بدلاً من ref.
    القواعس مرتبة حسب الأولوية:
    """
    
    refname = rem_file_ext(ref.name).lower()
    dupename = rem_file_ext(dupe.name).lower()
    
    # القاعدة 1: تفضيل الملفات بدون "نسخة" في الاسم
    if "نسخة" in dupename:
        return False  # dupe لديها "نسخة" → احتفظ بـ ref
    if "نسخة" in refname:
        return True   # ref لديها "نسخة" → بدل

    # القاعدة 2: تفضيل الأصلي على المكررات المرقمة
    # مثال: "صورة.jpg" مفضل على "صورة (1).jpg"
    if is_same_with_digit(dupename, refname):
        return False  # dupe نسخة مرقمة → احتفظ بـ ref
    if is_same_with_digit(refname, dupename):
        return True   # ref نسخة مرقمة → بدل

    # القاعدة 3: تفضيل الملفات في مجلدات أقل عمقاً
    # (المجلد الجذر مفضل على المجلدات المتداخلة)
    if len(dupe.path.parts) > len(ref.path.parts):
        return False  # dupe أعمق → احتفظ بـ ref
    
    return False  # خلاف ذلك احتفظ بـ ref الحالي
```

**أمثلة:**

| الملف المرجعي | الملف المكرر | الاختيار | السبب |
|---|---|---|---|
| `صورة.jpg` | `صورة (نسخة).jpg` | احتفظ بالمرجعي | لا "نسخة" في المرجعي |
| `أغنية (1).mp3` | `أغنية.mp3` | المكرر يصبح مرجعياً | المكرر هو الأصلي |
| `/صورة.jpg` | `/أرشيف/صورة.jpg` | احتفظ بالمرجعي | المرجعي أقل عمقاً |
| `مستند.pdf` (100 كيلوبايت) | `مستند.pdf` (100 كيلوبايت) | احتفظ بالمرجعي | نفس الأولوية |

الملفات: [

### التنفيذ خطوة بخطوة:

#### المرحلة 1: استخراج الكلمات

```
الملف 1: ["صورة", "إجازة", "2024"]
الملف 2: ["صورة", "إجازة", "2024", "معدلة"]
الملف 3: ["صورة", "شاطئ", "2024"]
```

#### المرحلة 2: بناء القاموس

```
القاموس: {
    "صورة": {الملف1, الملف2, الملف3},
    "إجازة": {الملف1, الملف2},
    "2024": {الملف1, الملف2, الملف3},
    "معدلة": {الملف2},
    "شاطئ": {الملف3},
}
```

#### المرحلة 3: مقارنة الملفات

**المقارنة 1: الملف 1 مقابل الملف 2 (عبر كلمة "صورة")**

```
كلمات الملف1: ["صورة", "إجازة", "2024"]
كلمات الملف2: ["صورة", "إجازة", "2024", "معدلة"]

المتطابقات: "صورة", "إجازة", "2024" (3 كلمات)
الإجمالي: (3 + 4) = 7 كلمات
النسبة المئوية: (3 × 2) / 7 × 100 = 85.7% ≈ 86%

النتيجة: Match(الملف1, الملف2, 86)  ✓ أعلى من عتبة 80%
```

**المقارنة 2: الملف 1 مقابل الملف 3 (عبر كلمة "صورة")**

```
كلمات الملف1: ["صورة", "إجازة", "2024"]
كلمات الملف3: ["صورة", "شاطئ", "2024"]

المتطابقات: "صورة", "2024" (كلمتان)
الإجمالي: (3 + 3) = 6 كلمات
النسبة المئوية: (2 × 2) / 6 × 100 = 66.7% ≈ 67%

النتيجة: Match(الملف1, الملف3, 67)  ✗ أقل من عتبة 80%
```

#### المرحلة 4: تشكيل المجموعات

```
المطابقات ≥ 80%: [Match(الملف1, الملف2, 86)]

المجموعات المشكلة:
├─ مجموعة 1: [الملف1, الملف2]
│  └─ المرجعي: الملف1 (أو الملف2 حسب قاعدة الربط)
│
└─ الملف3: لم يتطابق مع أي ملف (لا يظهر في النتائج)
```

#### المرحلة 5: النتيجة النهائية

```
المجموعة 1:
  └─ المرجعي: صورة_إجازة_2024.jpg (2.5 ميجابايت)
     └─ مكرر: صورة_إجازة_2024_معدلة.jpg (2.5 ميجابايت) [86% مطابقة]
```

## تحليل تعقيد الخوارزمية

### تعقيد الوقت

| العملية | التعقيد | الملاحظات |
|--------|--------|----------|
| استخراج الكلمات | O(N × M) | N ملف، M متوسط كلمات لكل ملف |
| بناء القاموس | O(N × M) | مسح خطي للملفات والكلمات |
| تقليل الشائع | O(W) | W كلمات فريدة |
| دمج متشابه | O(W² × log W) | فحوصات التشابه التربيعية |
| مقارنة الملفات | O(W × F²) | W كلمات، F ملفات لكل كلمة |
| تشكيل المجموعات | O(M) | M زوج مطابقة |
| **الإجمالي** | **O(W × F²)** | تسيطر عليها مقارنة الملفات |

### تعقيد الفضاء

| بنية البيانات | الفضاء | الملاحظات |
|---|---|---|
| قاموس الكلمات | O(W × F) | W كلمات، F ملفات لكل كلمة |
| أزواج مقارنة | O(F²) | تتبع جميع أزواج الملفات |
| المجموعات | O(N) | N ملف في المجموعات |
| **الإجمالي** | **O(W × F)** | يسيطر عليها القاموس |

### السيناريوهات الأفضل/الأسوأ

**الحالة الأفضل:** ملفات متطابقة (أحجام مختلفة)
- **الوقت:** O(1) لكل زوج (مقارنة حجم فقط)
- **مثال:** 10,000 ملف → 1 ثانية

**الحالة المتوسطة:** أسماء متشابهة
- **الوقت:** O(M × log M) حيث M = المطابقات الموجودة
- **مثال:** 10,000 ملف → 5-30 ثانية

**الحالة الأسوأ:** ملفات متشابهة جداً
- **الوقت:** O(N²) مقارنات
- **مثال:** 1,000 ملف نفس الحجم → 1-5 دقائق
- **التخفيف:** تقليل الكلمات الشائعة، تصفية حسب العتبة

## الملخص

خوارزمية كشف المكررات في **dupeGuru** هي **نظام متقدم متعدد المراحل** يقدم:

1. ✅ **دعم استراتيجيات مطابقة متعددة** (محتوى، اسم، بيانات وصفية)
2. ✅ **تحسين للأداء** (تجزئة متعددة المستويات، تصفية الكلمات)
3. ✅ **معالجة الحالات الخاصة** (ملفات ref، أنواع مختلطة، مسارات مكررة)
4. ✅ **تجميع ذكي** (مطابقة متعدية)
5. ✅ **اختيار مراجع ذكي** (الحفاظ على الأصليات على حساب النسخ)
6. ✅ **معالجة الأخطاء بشكل حسّن** (حدود الذاكرة، الملفات المفقودة)

الخوارزمية جاهزة للإنتاج وتم اختبارها على ملايين الملفات عبر أنظمة متنوعة (Linux, macOS, Windows).
